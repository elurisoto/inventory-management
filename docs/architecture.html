Global Filter System — useFilters.js

selectedPeriod
"all" | "2025-01"…"2025-12" | "Q1-2025"…"Q4-2025"
selectedLocation
"all" | "San Francisco" | "London" | "Tokyo"
selectedCategory
"all" | "Circuit Boards" | "Sensors" | "Actuators" | "Controllers" | "Power Supplies"
selectedStatus
"all" | "Delivered" | "Shipped" | "Processing" | "Backordered"
getCurrentFilters() maps composable refs to API param names: selectedLocation → warehouse   selectedCategory → category   selectedPeriod → month   selectedStatus → status All views import this composable and watch its refs — a filter change triggers a fresh API call in every mounted view.

API Endpoints — FastAPI (port 8001)

Method Path Description Filter Params
GET /api/inventory List all inventory items warehouse, category
GET /api/inventory/{item_id} Single inventory item detail —
GET /api/orders List orders with filtering warehouse, category, status, month
GET /api/orders/{order_id} Single order detail —
GET /api/dashboard/summary KPI aggregates (inventory value, low stock, pending, backlog) warehouse, category, status, month
GET /api/demand Demand forecast items (no filters) —
GET /api/backlog Backlog items; injects has_purchase_order flag —
GET /api/spending/summary Total procurement, operational, labor, overhead + % changes —
GET /api/spending/monthly Monthly cost breakdown by type (12 months) —
GET /api/spending/categories Category spend breakdown with percentages —
GET /api/spending/transactions Transaction ledger list —
GET /api/reports/quarterly Quarterly order counts, revenue, fulfillment rates —
GET /api/reports/monthly-trends Monthly order count, revenue, delivery count —
POST /api/purchase-orders Create purchase order for a backlog item —
GET /api/purchase-orders/{id} Get purchase order for a backlog item —
